# Docker 常用命令


## 一、镜像命令
| 命令                          | 作用             | 【重要注意事项】                                             |
| ----------------------------- | ---------------- | ------------------------------------------------------------ |
| `docker images`               | 查看本地所有镜像 | 直接运行，显示 **REPOSITORY（镜像名）** 和 **TAG（版本）**   |
| `docker pull 镜像:版本`       | 下载镜像         | 必须写 **镜像名+版本**，不写版本默认 latest                  |
| `docker build -t 名字:标签 .` | 构建镜像         | **最后必须加 .** 表示当前目录，必须在 Dockerfile 所在目录执行 |
| `docker rmi 镜像ID/名`        | 删除镜像         | 镜像被容器使用时**无法删除**，必须先删容器                   |
| `docker save 镜像 -o xxx.tar` | 导出镜像         | **前面是真实镜像名**，后面文件名**可随便写**                 |
| `docker load -i xxx.tar`      | 导入镜像         | 导入后**镜像名和版本会自动还原**，无需手动命名               |

## 二、容器命令
| 命令                             | 作用             | 【重要注意事项】                   |
| -------------------------------- | ---------------- | ---------------------------------- |
| `docker ps`                      | 查看运行中的容器 | 显示 **容器ID、容器名、端口**      |
| `docker ps -a`                   | 查看所有容器     | 包括**已停止**的容器               |
| `docker stop 容器ID/名`          | 停止容器         | 只能加 **容器ID / 容器名**         |
| `docker rm 容器ID/名`            | 删除容器         | 必须**先停止**才能删               |
| `docker exec -it 容器ID/名 bash` | 进入容器         | 必须加 **容器ID/名**，不能加服务名 |
| `docker logs 容器ID/名`          | 查看日志         | 只能看**指定容器**                 |

---

## 三、Docker Compose 命令（最容易错！）
| 命令                            | 作用             | 【⚠️ 超级重要注意事项】                                       |
| ------------------------------- | ---------------- | ------------------------------------------------------------ |
| `docker compose up -d`          | 后台启动所有服务 | **必须在 docker-compose.yml 所在目录执行**                   |
| `docker compose down`           | 停止并删除容器   | **不加任何参数！**<br>不加ID、不加名字，直接运行             |
| `docker compose ps`             | 查看compose容器  | 必须在 yml 目录下                                            |
| `docker compose logs 服务名`    | 查看服务日志     | **只能加 yaml 里的服务名**（如 mysql-db、python-app）<br>**不能加容器ID/容器名** |
| `docker compose restart 服务名` | 重启服务         | 同上，只能加 **服务名**                                      |

---

## 四、Docker 服务命令
| 命令                            | 作用       | 【重要注意事项】                       |
| ------------------------------- | ---------- | -------------------------------------- |
| `sudo systemctl start docker`   | 启动Docker | 必须加 sudo                            |
| `sudo systemctl restart docker` | 重启Docker | 修改 daemon.json 镜像源后**必须重启**  |
| `sudo systemctl daemon-reload`  | 重载配置   | 修改配置文件后**必须执行**，否则不生效 |

---

## 五、权限命令
| 命令                            | 作用             | 【重要注意事项】                                          |
| ------------------------------- | ---------------- | --------------------------------------------------------- |
| `sudo usermod -aG docker $USER` | 免sudo使用docker | **设置后必须重开终端 / 重启 / 执行 newgrp docker 才生效** |

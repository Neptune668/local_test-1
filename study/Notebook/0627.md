## 1.算法

```python
# 给定一个表示 大整数 的整数数组 digits，其中 digits[i] 是整数的第 i 位数字。
# 这些数字按从左到右，从最高位到最低位排列。这个大整数不包含任何前导 0。
# 将大整数加 1，并返回结果的数字数组
class Solution:
    def plusOne(self, digits: list[int]) -> list[int]:
        for i in range(len(digits) - 1, -1, -1):
            if digits[i] < 9:
                digits[i] += 1
                return digits
            digits[i] = 0

        return [1] + digits

sol = Solution()
print(sol.plusOne([1, 2, 9, 9]))
```

## 2.面试

多层装饰器的执行机制

### 1. 黄金法则（面试直接背这句）

- **叠加顺序（代码书写）**：靠近函数定义（`def`）的装饰器先执行。
- **执行顺序（数学等价）**：`@A @B def f()` 等价于 **`f = A(B(f))`**。
- **调用顺序（运行时）**：**自下而上**包装，**自上而下**调用（先进入最外层装饰器，最后进入原函数）。

------

### 2. 直观代码演示（打印日志法）

看下面这段代码，猜猜打印顺序是什么？

python

```
def decorator_a(func):
    print("1. 装饰器A 正在包装（外层）")
    def wrapper(*args):
        print("3. [调用] 进入A包装器")
        result = func(*args)
        print("6. [调用] 离开A包装器")
        return result
    return wrapper

def decorator_b(func):
    print("2. 装饰器B 正在包装（内层）")
    def wrapper(*args):
        print("4. [调用] 进入B包装器")
        result = func(*args)
        print("5. [调用] 离开B包装器")
        return result
    return wrapper

@decorator_a  # 外层
@decorator_b  # 内层
def my_add(x, y):
    print(">>> 原函数执行中 <<<")
    return x + y

# 执行调用
my_add(1, 2)
```



**输出结果（绝对让你一眼看穿）：**

text

```
1. 装饰器A 正在包装（外层）
2. 装饰器B 正在包装（内层）
3. [调用] 进入A包装器
4. [调用] 进入B包装器
>>> 原函数执行中 <<<
5. [调用] 离开B包装器
6. [调用] 离开A包装器
```



**解读：**
**定义时**：Python先读取最内层的`@decorator_b`（打印1），把原函数变成`wrapper_b`；再把`wrapper_b`传给外层`@decorator_a`（打印2）。
**调用时**：执行顺序像“剥洋葱”，**由外向内进（3→4），由内向外出（5→6）**。

### 3. 结合你的“缓存面试题”场景（实战意义）

假设你既有缓存装饰器，又有日志装饰器：

python

```
@logger          # 外层：记录调用日志
@cached(3)       # 内层：先检查缓存
def get_data(id):
    return 查询数据库...
```



- **定义时**：`cached`先包装原始函数，`logger`再包装`cached`返回的`wrapper`。
- **调用时**：
  1. 先进入`logger`（打印“正在调用get_data”）。
  2. 再进入`cached`（检查缓存，若命中直接返回，**根本不会进入原函数**）。
  3. 若缓存未命中，才执行原函数。

这种机制让你可以灵活组合功能，且**内层装饰器决定了外层的`func`到底是什么**。

------

### 4. 面试加分陷阱（一定要提）

面试官如果问你“有什么坑”，一定要答出这两点：

1. **`functools.wraps` 的重要性**：如果没有`@wraps(func)`，多层装饰后，原函数的`__name__`和`__doc__`会丢失（变成最外层`wrapper`的名字）。**在你的缓存题标准答案里，务必加上`@wraps(func)`**。
2. **参数传递**：外层`wrapper`接收`(*args, **kwargs)`，传给内层`wrapper`，再传给原函数。中间任何一层如果篡改了参数，后续所有层都会受影响。
